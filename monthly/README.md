# LF몰 제휴 월마감 분석 대시보드

일별 대시보드(`../app.py`)의 개념(총결제/순결제, 전년비 ▼상승·△하락, 목표비)을 승계한
**월마감/MTD 기준** 분석 앱. 차트 없이 표 중심, 4개 탭(개요·주차별·제휴사별·BPU별).

## 실행
```bash
cd monthly
streamlit run app.py
```

## 업로드 5종
| # | 파일 | 형식 |
|---|---|---|
| ① | 당년 raw (복수) | `유입실적_YY`·`인증회원_YY`·`인증거래액_YY`·`피벗` 시트 포함 월별 raw |
| ② | 전년 유입·인증 | ①과 동일 구조의 전년도 파일(UV·인증자수 사용) |
| ③ | 전년 매출분기 (복수) | ①과 동일 구조의 전년도 거래 파일 |
| ④ | 목표 | `target_template.xlsx` 양식 |
| ⑤ | LF몰 전체거래액 | `lfmall_template.xlsx` 양식 |

## 파생 로직 (피벗 수식 재현)
raw의 `당월인증`·`제휴사`·`거래액_VAT제외`·UV `제휴사명`은 **피벗 시트를 참조하는 엑셀 수식**이라,
파일에 계산 캐시가 없으면 비어 보인다. 앱은 그 수식을 파이썬으로 재현한다:
- **당월인증** = `정산일 YYYY-MM == 최초인증일 YYYY-MM`
- **제휴사** = 피벗 `<제휴사 구분>` 코드→약칭
- **거래액_VAT제외** = `거래액 ÷ 1.1`
- **UV 제휴사명** = AF코드 → 피벗 `<제휴사명 정제>` → `<제휴사 구분>`
- **주차/마감월** = 피벗 `<주차기준>` 일자→`YY_M_W`

## ⚠ DRM 주의
raw가 Softcamp 등 DRM으로 암호화(`SCDSA002`)된 경우 앱이 읽지 못한다.
**업로드 전 DRM 해제(사내 보안 해제)** 필요. 앱은 암호화 파일을 감지해 경고한다.

## 지표 정의
- 당월인증 CR = 당월인증고객수 ÷ 인증자수
- 객단가 = 거래액 ÷ 고객수 (당월인증/전체 각각)
- 거래액은 항상 VAT 제외. 총결제=판매만 / 순결제=판매+반품.
- LF몰 대비 비중 = (전체·당월인증)거래액 ÷ LF몰 실제
